I did a freaking language again. But this time I really REALLY like the results. 2025-08-21

So I created Spinach, a quantum language. Right now it can be translated to multiple other circuit representations that can be used directly on quantum computers. The idea is to create something more high-level representation. The code is shorter and more reusable. The two main advantages come from the ability to name things and the idea of pipelines. Spinach is somewhat inspired by functional programming. I don't think it deserves to be called functional. It just uses the pipelines like functions and everything turns around those pipelines. The pipelines are sequences of other pipelines or gates. In the end, they are sequences of gates. While doing some research about quantum computing I discovered that often, reversing a sequence of gates results in reversing their effects on a qubit and that property is used in some algorithms. So right now, I'm working on giving the ability to execute a pipeline in reverse. I would like to work eventually with researchers to see what they would like to see in my language and create a community around it. I just don't know how yet. The state of the project is a bit to pass from proof of concept to actual product. I want the tests and basic functionality to be solid before making it a community kind of project. I used Python with Lark. It's a great experience so far. The fact that it's Python makes it possibly compatible with Jupyter. Yes! I want the language to not only be its own thing but also be a Python library. A Python sublanguage for quantum. So someone will be able to use it to create a circuit that will be usable for simulation on Python. Isn't it fantastic! (Hakuna matata)
